Redes Neuronales: Teora y Prctica en Deteccin de Fraude Financiero con Autoencoders

Introduccin

La deteccin de fraude en transacciones financieras es un problema crucial en la era digital. A medida que los sistemas de pago electrnicos se expanden, tambin lo hacen las tcnicas fraudulentas, poniendo en riesgo los activos de millones de personas y empresas. Este proyecto se enfoca en la implementacin de tcnicas de aprendizaje automtico, con nfasis en redes neuronales profundas tipo autoencoder, para identificar transacciones sospechosas dentro de un conjunto realista de datos financieros.

Objetivos

Objetivo General:

Implementar un sistema de deteccin de anomalas basado en autoencoders utilizando Keras para identificar patrones de fraude financiero en un dataset simulado de transacciones bancarias.

Objetivos Especficos:

  • Analizar, explorar y preprocesar un conjunto de datos transaccionales con caractersticas numricas y categricas.
  • Desarrollar un modelo autoencoder para identificar transacciones anmalas sin supervisin directa.
  • Evaluar el modelo utilizando mtricas como precisin, recall y matriz de confusin.
  • Establecer umbrales ptimos de error de reconstruccin para maximizar la deteccin de fraudes reales.

Ncleo Problemtico

Las instituciones financieras enfrentan un constante desafo: detectar de manera precisa y en tiempo real transacciones fraudulentas entre millones de operaciones legtimas. La alta desproporcin de datos (desequilibrio de clases) y la naturaleza cambiante de los patrones de fraude dificultan el uso de enfoques tradicionales. Se requieren tcnicas capaces de aprender las caractersticas normales del sistema y detectar desviaciones sutiles que puedan representar riesgos.

Pregunta Problemtica

Es posible detectar transacciones financieras fraudulentas utilizando un autoencoder entrenado nicamente con datos normales, y cmo se puede ajustar su arquitectura y umbral de deteccin para maximizar el rendimiento del modelo?

Metodologa

  1. Exploracin del Dataset:
    Comprensin del conjunto de datos, estructura, tipos de variables y distribucin de clases.

  2. Preprocesamiento:

    • Limpieza de datos: manejo de valores nulos, duplicados y outliers.
    • Codificacin de variables categricas y normalizacin de datos numricos.

  3. Construccin del Autoencoder:

    • Arquitectura basada en capas simtricas de codificacin y decodificacin.
    • Entrenamiento exclusivamente con transacciones normales.

  4. Evaluacin del Modelo:

    • Clculo del error de reconstruccin sobre todo el dataset.

    • Evaluacin con mtricas de clasificacin.

  5. Visualizacin y anlisis de resultados:
    Anlisis grfico del error de reconstruccin y comparacin con etiquetas reales de fraude.

  6. Conclusin y reflexin:

def missing_values_per_column(df):
    """
    Esta funcin analiza y muestra informacin sobre los valores faltantes en un DataFrame de pandas.
    - Imprime el nmero de valores faltantes por columna.
    - Calcula y muestra el porcentaje de valores faltantes por columna.
    - Muestra el porcentaje total de valores faltantes en el DataFrame.
    - Genera un grfico de barras que visualiza el porcentaje de valores faltantes por columna.

    Argumentos:
    df -- DataFrame de pandas que se analiza para encontrar valores faltantes.

    Retorno:
    None
    """

    # Paso 1: Encontrar columnas con valores faltantes y calcular la cantidad de valores faltantes por columna
    faltantes_por_columna = df.isnull().sum()
    
    # Paso 2: Filtrar las columnas que tienen valores faltantes y convertir los resultados a un diccionario
    data1 = faltantes_por_columna[faltantes_por_columna > 0]
    variables_con_faltantes = data1.to_dict()
    
    # Imprimir la cantidad de valores faltantes por columna en formato JSON con sangra para mejor lectura
    print('\nCant. valores faltantes por columna:',
          json.dumps(variables_con_faltantes, indent=4),
          sep="\n")
    
    # Paso 3: Calcular el porcentaje de valores faltantes por columna y formatearlo como porcentaje
    data = (data1 / len(df)) * 100
    data = data.to_dict()
    porcentaje_faltantes_por_columna = {key: f'{value:.2f}%' for key, value in data.items()}
    
    # Paso 4: Calcular el porcentaje total de valores faltantes en todo el DataFrame
    porcentaje_total_faltantes = (df.isnull().sum().sum() / df.size) * 100
    porcentaje_total_no_faltantes = (df.notnull().sum().sum() / df.size) * 100
    
    # Imprimir el porcentaje de valores faltantes por columna en formato JSON con sangra
    print('\n% valores faltantes por columna:', f'{json.dumps(porcentaje_faltantes_por_columna, indent=4)}', sep="\n")

    # Crear un grfico de barras para visualizar el porcentaje de valores faltantes por columna
    plt.figure(figsize=(6, 3))
    plt.bar(variables_con_faltantes.keys(), variables_con_faltantes.values(), color=['blue', 'orange', 'green'])
    plt.title('Porcentaje de datos faltantes por columna')
    plt.ylabel('Porcentaje')
    plt.show()
    
    # Imprimir el porcentaje total de valores faltantes y no faltantes en el DataFrame
    print(f"\nPorcentaje total de valores faltantes en la base de datos: {porcentaje_total_faltantes:.2f}%")
    print(f"Porcentaje total de valores en la base de datos: {porcentaje_total_no_faltantes:.2f}%")

Autoencoder con Keras para Deteccin de Anomalas

Deteccin de anomalas no supervisado

La deteccin de anomalas no supervisada es una tcnica clave en el anlisis de datos que permite identificar patrones atpicos o inusuales sin necesidad de datos etiquetados. A diferencia de los mtodos supervisados, que requieren ejemplos previos de anomalas, este enfoque se basa en algoritmos que aprenden la estructura normal de los datos y destacan las desviaciones significativas. Es especialmente til en aplicaciones como fraudes, fallos en equipos o seguridad, donde las anomalas son raras o desconocidas. Entre los mtodos ms comunes se encuentran el clustering, los modelos estadsticos y el aprendizaje automtico basado en distancias o densidades.

1. Preprocesamiento para Autoencoder

En el contexto de la deteccin de fraude financiero, el reprocesamiento de datos es esencial para optimizar el rendimiento del autoencoder, dado el desbalance tpico de los conjuntos de datos (donde las transacciones fraudulentas son mnimas comparadas con las legtimas). Para garantizar que el modelo aprenda eficientemente los patrones normales de las transacciones, se aplican tcnicas como la normalizacin o estandarizacin de caractersticas, evitando que variables con magnitudes dispares (como montos o frecuencias de transacciones) dominen el entrenamiento.

2. Construccin del Autoencoder

El autoencoder implementado sigue una arquitectura simple pero efectiva, diseada para manejar las caractersticas especficas de los datos financieros. Con una capa de entrada que acepta dimensiones igual al nmero de features (definido por input_dim = X_train_scaled.shape[1]), el modelo reduce la dimensionalidad a solo 4 neuronas en la capa codificada (encoding_dim = 4), utilizando la funcin de activacin ReLU para introducir no linealidad y evitar valores negativos. La capa decodificada reconstruye la entrada original con activacin sigmoid, normalizando las salidas entre 0 y 1 para facilitar la interpretacin de los errores de reconstruccin. El modelo se compila con el optimizador Adam y la funcin de prdida Mean Squared Error (MSE), que penaliza las diferencias entre la entrada y la reconstruccin. Esta configuracin permite que el autoencoder aprenda eficientemente los patrones normales de las transacciones, identificando como anomalas aquellas operaciones con mayores errores de reconstruccin, lo que es particularmente til para detectar fraudes en datos altamente desbalanceados.

# Parmetros del autoencoder
input_dim = X_train_scaled.shape[1]
encoding_dim = 4  # Puedes ajustar este valor para ver diferentes resultados

input_layer = Input(shape=(input_dim,))
encoded = Dense(encoding_dim, activation='relu')(input_layer)
decoded = Dense(input_dim, activation='sigmoid')(encoded)

autoencoder = Model(inputs=input_layer, outputs=decoded)
autoencoder.compile(optimizer='adam', loss='mean_squared_error')

3. Entrenamiento del Autoencoder

El modelo de autoencoder se entren durante 50 pocas utilizando un batch size de 64 muestras, optimizando la funcin de prdida MSE mediante el algoritmo Adam. Para garantizar un entrenamiento robusto, se implement una divisin del 20% del conjunto de entrenamiento como validacin (validation_split=0.2), permitiendo monitorear el desempeo del modelo en datos no vistos durante el proceso de aprendizaje. El parmetro shuffle=True asegur que los datos se mezclaran antes de cada poca, evitando sesgos en el entrenamiento.

La grfica resultante muestra la evolucin de la prdida tanto en entrenamiento como en validacin, donde se observa una disminucin consistente en ambas mtricas a lo largo de las pocas. Esta convergencia estable indica que el modelo est aprendiendo efectivamente la representacin subyacente de las transacciones normales sin presentar sobreajuste. La capacidad del autoencoder para minimizar el error de reconstruccin en datos de validacin sugiere que podr generalizar adecuadamente a nuevas transacciones, aspecto crucial para identificar patrones fraudulentos que se desvan significativamente del comportamiento normal aprendido.

4. Evaluacin y Deteccin de Anomalas

El proceso de deteccin de anomalas se realiz calculando el Error Cuadrtico Medio (MSE) entre las transacciones originales y sus reconstrucciones generadas por el autoencoder. Se estableci un umbral de decisin en el percentil 95 de los errores de reconstruccin (threshold = np.percentile(mse, 95)), clasificando como potenciales fraudes aquellas transacciones con MSE superior a este lmite. Esta seleccin del percentil 95 permite enfocarse en las anomalas ms significativas mientras se controla la tasa de falsos positivos.

La evaluacin del modelo se complet mediante mtricas de clasificacin, incluyendo matriz de confusin y un reporte completo (precision, recall, F1-score), que comparan las predicciones con las etiquetas reales de fraude (isFraud). Estos resultados permiten analizar el equilibrio entre la capacidad del modelo para detectar transacciones fraudulentas (recall) y su precisin al minimizar falsas alarmas, aspectos crticos en aplicaciones financieras donde ambos factores tienen consecuencias operacionales importantes. La seleccin del umbral puede ajustarse segn los requisitos del negocio, priorizando mayor sensibilidad o precisin segn sea necesario.

  • Recall clase 1 (fraude): 23%, bastante bajo.

Esto sugiere que:

Posibles causas:

  • El conjunto de entrenamiento fue insuficiente o muy homogneo.
